# Stillwater POI sources

The map overlay `../stillwater-poi.json` is **generated** by merging the source
files in this directory. Don't hand-edit `stillwater-poi.json` — edit a source and
re-bake. See `scripts/bake-stillwater-poi.mjs`.

## The three sources (ascending precedence — later wins)

| File                                          | Source                                                  | You edit it?                            |
| --------------------------------------------- | ------------------------------------------------------- | --------------------------------------- |
| `stillwater-poi-osm.json`                     | Auto-baked from OpenStreetMap (Overpass). The fallback. | No — regenerated by `bake:poi:osm`.     |
| `stillwater-poi-google.geojson` **or** `.kml` | A Google My Maps export (optional).                     | Drop the export in as-is.               |
| `stillwater-poi-manual.json`                  | Hand-curated overrides you own.                         | **Yes** — add anything OSM/Google miss. |

When two entries refer to the same place — same `kind` and either the same
location or the same name within ~150 m — the higher-precedence source wins. So a
`manual` entry corrects a Google entry, which corrects an OSM entry. Distinct
places that merely share a name (opposite-side bus stops, two campuses) are kept
separate by the proximity check.

### Per-kind policy

`scripts/lib/poi-sources.mjs` defines `DEFAULT_MERGE_POLICY`:

- **Bus stops and restaurants come from Google (+ manual) only.** The curated My
  Maps bus stops carry route numbers and colors (e.g.
  `"Hester & 4th South (157) (Scarlet)"`) that OSM lacks and names differently, so
  OSM stops would only add worse-named near-duplicates. Restaurants are Google-only
  because OSM packs many distinct restaurants into the same strip mall — a location
  merge would wrongly fuse neighbours (Chico's + Five Guys). OSM entries for these
  two kinds are dropped.
- Bus stops are **not** location-deduped: each Google stop is uniquely numbered and
  real north/south & east/west pairs sit <30 m apart.
- **Schools and parks merge from all sources, deduped within ~50 m.** Both sources
  list the same place under slightly different names ("Westwood Elementary" vs
  "…School"; every park has a "…Playground" twin). Proximity dedup collapses the
  twins (Google wins) while keeping any OSM-only school/park the curated map lacks.
- Bare `Point N` scratch pins My Maps leaves behind are dropped (`dropName`).
- **Closed POIs** whose My Maps pin won't delete (a known Android issue) are dropped
  by exact name via the `CLOSED_POIS` list in `scripts/lib/poi-sources.mjs`. Add a
  name there when a place has closed but its pin lingers; it's harmless once the pin
  is actually removed.

The policy lives in `DEFAULT_MERGE_POLICY` in `scripts/lib/poi-sources.mjs`. The
bake log prints `dropped: N` — the count excluded by the source policy.

`kind` must be one of: `bus-stop`, `restaurant`, `school`, `park`. Coordinates are
GeoJSON order: `[longitude, latitude]`.

## Adding the missing restaurants

Fastest path — edit `stillwater-poi-manual.json`, add a feature per restaurant:

```json
{
  "type": "Feature",
  "properties": { "kind": "restaurant", "name": "Eskimo Joe's" },
  "geometry": { "type": "Point", "coordinates": [-97.0632, 36.1201] }
}
```

Then `npm run bake:poi` and the restaurant shows up on the map.

Or, for visual editing: build a Google My Maps map with one **layer per category**
(a "Restaurants" layer → `kind: restaurant`), export it as GeoJSON or KML, save it
here as `stillwater-poi-google.geojson`/`.kml`, and `npm run bake:poi`.

## Commands

```bash
npm run bake:poi:osm     # refetch OpenStreetMap → stillwater-poi-osm.json (network)
npm run bake:poi:google  # refetch the curated My Maps KML → stillwater-poi-google.kml (network)
npm run bake:poi         # merge all present sources → ../stillwater-poi.json (offline)
npm run bake:poi:all     # all three, in order (network)
```

`bake:poi:google` pulls the "Jet Lag: Stillwater" My Maps map (id in
`scripts/bake-stillwater-poi.mjs`, override with `GOOGLE_MID=<mid>`). The map must
stay shared as **"anyone with the link can view"** for the export to work. So the
edit-your-map loop is: edit in My Maps → `npm run bake:poi:google` → `npm run
bake:poi`. Re-fetching a source never touches your manual file.
